# Report Template

Use this structure for final answers.

## Assumptions
Briefly state inferred constraints.

## Search Strategy
Summarize query families and surfaces checked.

## Best Pick
Name one best starting point and explain why.

## Scenario-Based Shortlist
Use buckets such as best overall, fastest validation, best self-hosted, best library/API, best full app, mature option, lightweight option, promising but risky.

## Evidence Table
Columns: repo, fit, health, docs/setup, license, risk, confidence, why it matches.

## Candidate Details
For each strong candidate: evidence, gaps, integration notes, and validation effort.

## Rejected Or Lower-Fit Candidates
Mention relevant-looking candidates that were excluded and why.

## Key Risks
List blockers such as license, stale maintenance, setup complexity, missing feature, or open-core limitation.

## 30-60 Minute Validation Plan
Give concrete next steps for the top 1-3 repositories.

## Confidence
State whether the answer is metadata-level, README-level, issue-level, or execution-level.
